# ArteOdyssey デザイン原則

## コアコンセプト

**「静謐で落ち着いた、余白を活かしたUI」**

k8oの個人Webサイト開発のためのUIライブラリ。アクセシビリティとパフォーマンスを前提に、ミニマルで統一感のあるUIを提供する。

---

## デザイン原則

### 1. 引き算の美学
装飾は最小限に。要素を減らし、本当に必要なものだけを残す。

- グラデーションやシャドウは控えめに
- アイコンや装飾は意味があるときだけ使う
- 情報の優先度を明確にし、不要な要素は削る

### 2. 余白で語る
詰め込まず、空間にゆとりを持たせて読みやすさを確保する。

- コンポーネント間には十分なスペースを取る
- 一画面の情報量は少なめに、スクロールを許容する
- テキストの行間もゆったりと

### 3. 静かな変化
アニメーションは控えめに。派手な動きより、繊細なフィードバックを。

- トランジションは150ms以下を基本とする
- 必要なフィードバックのみ動きで表現
- bounce、spring系のアニメーションは避ける

### 4. 穏やかな色
目に優しいトーン。コントラストは必要十分に。

- 彩度は控えめに
- グレー系を効果的に活用
- 状態変化は透過度で表現することが多い

## スタイルトークン

### 角丸
控えめな丸みで、柔らかさを出しつつシャープさも残す。

| 用途 | トークン | 値 |
|------|----------|-----|
| 小さい要素（Badge等） | `rounded-sm` | 0.375rem |
| 標準（Button、Input等） | `rounded-md` | 0.5rem |
| 大きい要素（Card、Modal等） | `rounded-lg` | 0.75rem |

### シャドウ
基本的には使わない。使う場合も薄く。

| 用途 | 使用 |
|------|------|
| Card | `shadow-sm` または border のみ |
| Modal/Dialog | `shadow-md` |
| Dropdown | `shadow-sm` |
| Button | なし |

### 余白
ゆったりとした余白を基本とする。

| 用途 | サイズ |
|------|--------|
| コンポーネント内部 | `p-4` 〜 `p-8` |
| テキスト間 | `mt-2` 〜 `mt-4` |
| セクション間 | `mt-8` 〜 `mt-12` |

### ボーダー
細めで控えめな色を使用。

| 用途 | スタイル |
|------|----------|
| 区切り線 | `border-border-mute` |
| Input | `border border-border-base` |
| フォーカス | `ring-2 ring-border-info` |

### フォントウェイト
細め〜標準を基本とする。

| 用途 | ウェイト |
|------|----------|
| 本文 | `font-normal` (400) |
| 強調 | `font-medium` (450) |
| 見出し | `font-bold` (700) |

### トランジション
控えめで自然な動き。

| 用途 | 設定 |
|------|------|
| ホバー | `duration-150 ease-out` |
| フェード | `duration-200 ease-out` |
| スライド | `duration-150 ease-out` |

## カラーパレット

### ブランドカラー
- **Primary**: Teal（落ち着きと信頼感）
- **Secondary**: Cyan（アクセント）

### セマンティックカラー
- **Success**: Green
- **Warning**: Yellow
- **Error**: Red
- **Info**: Blue

### 使用指針
- 背景は `bg-base`（白/黒）を基本に
- 強調には `bg-subtle` や `bg-mute` を使用
- 原色は控えめに、状態表示やアクセントのみに使用

---

## コンポーネント設計指針

### Do（推奨）
- シンプルなプロップ設計
- セマンティックなHTML要素の使用
- キーボード操作への対応
- 適切なARIA属性の付与

### Don't（避ける）
- 過度な装飾やアニメーション
- 複雑なネスト構造
- 独自のフォーカススタイル（ブラウザ標準を活かす）
- 情報の詰め込み
